## About

**Breezy Build** is a subscription-based platform delivering **5-minute daily insights** on AI development tools. We simplify the overwhelming landscape of AI tools like Claude, Cursor, v0, Bolt, and ChatGPT for founders who want to build products without traditional coding.

### Why Breezy Build?

- **Curated Daily Content** - No more doom-scrolling through AI news. Get exactly what matters.
- **Non-Technical Focus** - Explanations designed for founders, not developers.
- **Actionable Insights** - Every piece of content includes practical takeaways.
- **Community Driven** - Join founders building real products with AI.

## Getting Started

### Prerequisites

- **Node.js** 16.x or higher
- **npm** or **yarn**
- **Supabase** account
- **Stripe** account

### Installation

```bash
# Clone the repository
git clone https://github.com/yourusername/breezybuild.git

# Navigate to project directory
cd breezybuild

# Install dependencies
npm install
```

### Environment Variables

Create a `.env.local` file in the root directory:

```env
# Supabase
NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key

# Stripe
NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=your_stripe_publishable_key
STRIPE_SECRET_KEY=your_stripe_secret_key
STRIPE_WEBHOOK_SECRET=your_webhook_secret

# App
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

### Run Development Server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

## API Routes

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/checkout` | POST | Create Stripe checkout session |
| `/api/stripe/webhook` | POST | Handle Stripe webhooks |
| `/api/subscribe` | POST | Email subscription |
| `/api/tools` | GET | Fetch tool data |
| `/api/tools/[id]` | GET | Get specific tool |

## Database Schema

The application uses Supabase (PostgreSQL) with the following core tables:

- **profiles** - User profile data
- **subscriptions** - Active subscriptions
- **payments** - Payment history
- **newsletters** - Newsletter content
- **newsletter_subscribers** - Subscriber management
- **subscription_tiers** - Pricing tier definitions

Full schema documentation: [`DATABASE_SCHEMA.md`](./DATABASE_SCHEMA.md)

## Deployment

### Netlify (Recommended)

1. Connect your repository to Netlify
2. Configure environment variables in Netlify dashboard
3. Set build command: `npm run build`
4. Set publish directory: `.next`

### Stripe Webhooks

Configure webhooks in Stripe Dashboard pointing to:
```
https://yourdomain.com/api/stripe/webhook
```

Required events:
- `checkout.session.completed`
- `customer.subscription.created`
- `customer.subscription.updated`
- `customer.subscription.deleted`
- `invoice.payment_succeeded`
- `invoice.payment_failed`

See [`STRIPE_WEBHOOK_SETUP.md`](./STRIPE_WEBHOOK_SETUP.md) for detailed instructions.
